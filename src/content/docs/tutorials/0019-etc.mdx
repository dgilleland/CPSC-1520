## `grok-decisions.js`

Decisions affect what parts of our code execute at run-time. They are the way we are able, as developers, to *choose* what we want our program to do. All of this can be accomplished with the humble `if` statement.

### Basic Grammar

The first thing to understand about the `if` statement is its *grammar* [TODO: Link]. The basic grammar looks like this.

```js
if(conditionalExpression)
  statementOrStatementBlock // true side
else
  statementOrStatementBlock // false side
```

There are two keywords in this grammar.

- The **`if`** keyword tells the computer we're about to make a true/false decision based on some information (the *`conditionalExpression`*). The *`statementOrStatementBlock`* will be executed only if the conditional expression produces a `true` boolean result.
- The **`else`** keyword tells the computer what to do should that conditional expression turns out to produce a `false` result. Note that the `else` portion with its `statementOrStatementBlock` is entirely optional.

Besides the keywords, we have these *placeholder* parts of the grammar.

- The ***`conditionalExpression`*** is simply any *expression* [TODO: link to Statements and Expressions] that can produce a `true` or `false` value.
  - *By the way, the words `true` and `false` are JavaScript keywords.*
- A ***`statementOrStatementBlock`*** is either a **single** statement or a **single** statement ***block*** (a block is zero or more statements inside curly braces - `{ }`).
  - A single statement can be on its own line.
  - A statement block should have the opening curly brace at then end of the same line as the initial statement and the closing curly brace on its own line.

The following is an example of a statement block with properly positioned curly braces.

```js title="Sample Code"
if(quantity === 0) {
  // zero or more lines of code
} else {
  // zero or more lines of code
}
```


It's important to remember that this **if-else** structure provides **alternate paths of logic**. If one path is followed, then the other is not followed.


### Part 1 - Your First If/Else

Let's explore the If/Else statement with a few examples. I recommend also running these in debug mode.

<Steps>
1. To begin, create a file called `grok-decisions.js`. Then run it in the terminal with `node --watch grok-decisions.js`.

    <Aside title="Grok">
    If you're curious about the word "grok", check out this <a href="https://grokmag.com/grok-definition/" target="_blank">bit of history</a>.
    </Aside>

1. Let's add some code to portray using a self-serve gas station where we want to fuel up our vehicle.

    ```js title="grok-decisions.js"
    let bankBalance = 78.23;
    let preAuthorizationAmount = 50.00;

    if(bankBalance > preAuthorizationAmount) {
      console.log('Transaction Approved.');
      console.log('Please Remove Card Now and Lift Nozzle');
    }
    ```

1. We've hard-coded values in our program, and you can probably guess what is going to be displayed on the screen. Because the bank balance is the larger of the two values, then the conditional expression `bankBalance > preAuthorizationAmount` results in a `true`, and then the code inside statement block is executed.

    If you're using the debugger, try placing a breakpoint on the `if` statement and then step through the code.

1. Let's edit the `preAuthorizationAmount` and increase it to $100.

    ```diff lang="js" title="grok-decisions.js"
    - let preAuthorizationAmount = 50.00;
    + let preAuthorizationAmount = 100.00;
    ```

1. Now, because $78.23 is *not* greater than $100.0, the code execution jumps past the instructions on the *"true"* side of the `if` statement, and nothing is displayed on the screen.

1. Let's add an `else` side to our `if` statement.

    ```diff lang="js" title="grok-decisions.js"
      if(bankBalance > preAuthorizationAmount) {
        console.log('Transaction Approved');
        console.log('Please Remove Card Now and Lift Nozzle');
    - }
    + } else {
    +   console.log('Transaction Declined.');
    +   console.log('Please Remove Your Card').;
    + }
    ```

1. Play with making changes to either the `bankBalance` or the `preAuthorizationAmount` and observe the effects. Can you control which output is generated by modifying the values of these variables?

    <Aside title="A bug, or a feature? 🐞">
    What would happen if both variables have the *same amount*? Is that what you think *should* happen? What changes would you want to try with this sample code?
    </Aside>
</Steps>

With this simple example, a whole world of possibilities begins to open up for us as programmers. You will use [Flow-Control] statements just like these throughout your career.

### Conditional Expressions




Conditional expressions are ones that are ultimately evaluated as being equivalent to `true` or `false`. In flow-control statements, they often include the use of relational operators and/or boolean operators. That's because we often want to base our decision on some *comparison* between  pieces of information. But how do we compare things? Through the use of some important ***operators***.

### Relational Operators

JavaScript has the following relational operators. These are all binary operators, meaning that there must be some identifier or expression on both sides of the operator

- `==` - "Is equal to"
- `===` - Strict equality
- `!=` - "Not equal to"
- `!==` - Strict inequality
- `>`- Greater Than
- `>=` - Greater Than or Equal To
- `<`- Less Than
- `<=` - Less Than or Equal To

## Part 4 - Complex Descisions

We can combine arithmetic, relational and boolean operations in a conditional expressions. When we do that, the natural order of those operations follows this precidence.

1. Arithmetic operations are executed to produce some value.
2. Relational operations are performed next to compare values and produce some *boolean* value.
3. Boolean operations/comparisons are evaluated to produce a `true`/`false` result.
